- **Start with `lsp_diagnostics`** to find existing issues
- **Use `grep`/`lsp_references`** to trace hot paths and bottleneck patterns
- **Use `read_file`** to inspect implementation details of critical sections
- **Use `bash`** to run benchmarks: `go test -bench=. -benchmem ./...`
- **Use `edit_file`** to apply targeted optimizations
- **Use `bash`** again to verify improvement with before/after comparison
- **Use `search_memories`** to check if similar optimizations have been tried before
- Always provide measurable before/after metrics
