# Agent Coordination Patterns

## Parallel Analysis Pattern
When facing complex problems requiring multiple perspectives:
```
Phase 1: Parallel Analysis
- Athena: Strategic assessment
- Artemis: Technical feasibility
- Hestia: Security implications

Phase 2: Integration
- Hera: Synthesize findings
- Eris: Resolve conflicts

Phase 3: Documentation
- Muses: Record decisions
```

## Security-First Pattern
For security-critical operations:
```
1. Hestia: Initial security scan
2. Artemis + Athena: Parallel impact assessment
3. Eris: Coordinate mitigation plan
4. Muses: Document security measures
```

## Performance Optimization Pattern
For optimization tasks:
```
1. Artemis: Profile and identify bottlenecks
2. Hestia: Security impact analysis
3. Athena: Architectural review
4. Hera: Orchestrate implementation
```

## Emergency Response Pattern
For critical incidents:
```
Immediate: Eris coordinates response
Parallel:
- Artemis: Technical fix
- Hestia: Security patch
- Athena: Communication strategy
- Hera: Resource allocation
```

## Agent Selection Logic
| Task Type | Primary Agent | Support Agents |
|-----------|---------------|----------------|
| System Design | Athena | Artemis, Hestia |
| Optimization | Artemis | Hestia, Athena |
| Security Audit | Hestia | Artemis, Eris |
| Team Coordination | Eris | All agents |
| Strategic Planning | Hera | Athena, Muses |
| Documentation | Muses | Relevant agents |

## Communication Protocols
### Direct Request
Agent A → Agent B: Specific expertise request

### Broadcast
Agent → All: Status updates or alerts

### Cascade
Agent A → B → C: Sequential processing

## Conflict Resolution
When agents disagree:
1. Eris mediates conflicts
2. Priority: Security > Stability > Performance > Features
3. Hera makes final strategic decisions
4. Document rationale via Muses
