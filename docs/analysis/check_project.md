# 実行タスク
提供されたコンテキストに基づき、以下の分析ステップを厳密に実行してください。

1. **実装の完成度と重複の検証 (最優先)**
    - **未実装・モックの検出:** `TODO`、`FIXME`、`pass`のみ、あるいはコメントだけで中身がない関数やクラスを特定してください。
    - **車輪の再発明の検出:** 標準ライブラリやプロジェクト内の既存ユーティリティで解決できる処理を、独自に実装している箇所を指摘してください。
    - **似た処理の散見:** DRY原則（Don't Repeat Yourself）に違反している重複ロジックを特定してください。

2. **アーキテクチャと設計の評価**
    - **関心の分離 (SoC):** ファイルやクラスの責務は単一か、モジュール間の結合度は適切（疎結合）か。
    - **ディレクトリ構造:** ファイルの配置は直感的かつ論理的か。
    - **デザインパターン:** 適切なパターンが使用されているか、逆に過剰なエンジニアリング（YAGNI違反）になっていないか。

3. **コードの複雑性と保守性 (定量的な兆候の分析)**
    - **複雑度の高いロジック:** ネストが深すぎる（目安: 3階層以上）、条件分岐が多すぎる関数を特定してください（厳密な数値計算は不要ですが、リスクが高い箇所を指摘すること）。
    - **可読性:** 変数名・関数名は「何をするか」を明確に示しているか。マジックナンバーやハードコーディングはないか。

4. **言語固有のベストプラクティスと安全性**
    - **イディオム:** そのプログラミング言語特有の効率的な書き方（Pythonic, Modern JSなど）ができているか。
    - **古い慣習:** 推奨されない古い構文やライブラリが使われていないか。
    - **セキュリティ:** SQLインジェクション、XSS、機密情報のハードコーディング等の基本的な脆弱性兆候。
    - **エラーハンドリング:** エラーを握りつぶしていないか（空のcatch）、リソース解放は適切か。

5. **総合評価と改善ロードマップ**
    - 全体の品質スコア（100点満点）と、修正インパクトが大きい順に3つの優先タスクを提案してください。
